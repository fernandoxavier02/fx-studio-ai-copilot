# Superpowers gates
- Se o trabalho segue o workflow `superpowers`, não pule brainstorming quando ainda faltarem framing, restrições ou critérios claros.
- Não avance para execução sem plano explicitamente aprovado.
- Durante execução, mantenha checkpoints por batch com validação local antes de avançar.
- Não declare conclusão final sem verificação explícita do que foi prometido versus o que foi entregue.
- Quando o runtime não enforce o gate sozinho, trate isso como compatibilidade guiada e diga isso claramente.
